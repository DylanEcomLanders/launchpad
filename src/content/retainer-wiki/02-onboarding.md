# Onboarding & Audit

## Day 1-3: The Deep Dive

### Day 1: Data Collection
- Access granted: GA4, Shopify analytics, heatmaps (Hotjar/MS Clarity), ad accounts
- Run full site through conversion matrix
- Screenshot key pages (homepage, top 3 PDPs, cart, checkout)
- Pull traffic sources, device split, funnel drop-off data

### Day 2: Audit & Analysis
- Score each funnel layer (traffic → checkout)
- Identify top 3 revenue leaks with estimated impact
- Review competitor post-click experiences
- Document offer/positioning gaps
- Assess current testing maturity (are they testing at all?)

### Day 3: Roadmap Build
- Prioritise page builds by revenue impact (not client preference)
- Create 90-day roadmap with monthly milestones
- Build design briefs with section-by-section breakdown
- Draft testing roadmap with if/then hypothesis chains
- Prepare onboarding deck for client kickoff

## Client Kickoff Meeting

**Agenda:**
1. Audit findings walkthrough (the problem)
2. Roadmap presentation (the plan)
3. Month 1 deliverables confirmed (the action)
4. Communication rhythm agreed (Slack, weekly updates, monthly call)
5. Access and assets checklist

## What We Need From the Client

- [ ] GA4 viewer access
- [ ] Shopify analytics access (or data export)
- [ ] Brand guidelines (logo, fonts, colours)
- [ ] Product photography / assets folder
- [ ] Top 3 products by revenue
- [ ] Current ad creatives (or Meta Ad Library access)
- [ ] Any existing research (customer surveys, reviews, NPS)
- [ ] Key stakeholder for approvals (one person, not a committee)

## Onboarding Red Flags

Watch for these early — they predict churn:
- Multiple approval stakeholders ("the whole team needs to see it")
- No analytics access ("we'll send you screenshots")
- Vague goals ("just make it better")
- Price sensitivity in Month 1 ("can we reduce scope this month?")
- No ad spend ("we're pausing ads for now") — no traffic = no testing = no results
