# Delivery Framework

## The Core Principle

**We control the strategy. The client trusts the system.**

This is not a vendor relationship where the client briefs pages and we execute. We own the conversion roadmap. We decide what gets built, when, and why — based on data, not client preference. The client approves direction, not deliverables.

This is what justifies £8K+/mo. They're not buying pages. They're buying a team that obsesses over their conversion system every single month and compounds results over time.

---

## First 90 Days — The Conversion System Build

The first 90 days follow a sequential 5-phase delivery. Each phase builds on the last. This is how an audit becomes compounding revenue.

### Phase 1: Discovery Audit (Pre-Sale)
Broad funnel review using public data + initial conversation intel. Score every layer of the conversion matrix. Identify the 3 biggest revenue leaks. This happens during the two-stage sales process — it's what sells the partnership.

### Phase 2: Deep Audit (Week 1)
Once we have backend access (GA4, Shopify analytics, heatmaps), go deep. Validate or adjust the discovery scores with real data. This is where estimates become facts. Actual funnel drop-off rates, device-level conversion splits, page-level performance, AOV breakdown, post-purchase flow analysis.

### Phase 3: Visual Roadmap (Week 1-2)
Build the 60-90 day visual roadmap on Miro. Prioritise the fixes: pages to build, offers to restructure, tests to run, post-purchase flows to create. Each item is a "conversion asset" — not a deliverable, but a revenue lever. Roadmap is sequenced by impact (ICE scoring), not by what's easiest or what the client wants.

### Phase 4: Build & Ship (Weeks 2-12)
Every conversion asset gets:
- A design brief (section-by-section breakdown, hypothesis, success metric)
- Design → client review → development → QA gate
- Nothing goes live without passing QA
- Builds include: landing pages, PDP redesigns, cart/checkout optimisation, bundle pages, post-purchase upsell pages, email flow templates

### Phase 5: Measure & Compound (Ongoing)
After launch, set up tracking for each asset. Measure against the hypothesis. If it worked → iterate and push further. If it didn't → diagnose why, test the alternative. This is the compounding engine — every asset feeds data back into the next cycle.

> **The 5-phase system IS the product.** If the pipeline (audit → score → roadmap → brief → build → measure) doesn't flow smoothly, everything stalls. The roadmap is the engine. Without it, you're back to ad-hoc client requests.

## Monthly Delivery Rhythm (Post-90 Days)

Once the initial 90-day system is in place, delivery follows a consistent monthly cadence:

### Week 1: Strategy & Planning
- Review last month's test results and page performance
- Update the visual roadmap based on data
- Write design briefs for this month's builds
- Async update to client: "Here's what we're building this month and why"

### Week 2-3: Design & Build
- Design pages per brief (section-by-section, hypothesis-driven)
- Client review round (ONE round — not endless revisions)
- Development and build
- QA gate before anything goes live

### Week 4: Launch & Measure
- Push live
- Set up tracking and monitoring
- Document baseline metrics for comparison
- Plan next month's priorities based on this month's data
- Monthly report delivered to client

## Design Brief Template

Every conversion asset gets a brief before design starts. No brief = no build.

1. **Asset type** — Landing page, PDP, cart redesign, bundle page, post-purchase page
2. **Traffic source** — Where are visitors coming from? (Meta, Google, email, organic)
3. **Objective** — What's the one job this page needs to do? (convert cold traffic, increase AOV, drive upsell)
4. **Audience** — Who lands here? Cold traffic? Retargeting? Existing customers?
5. **Section-by-section breakdown** — Hero, social proof, features, objection handling, comparison, CTA
6. **Hypothesis** — "We believe [change] will [outcome] because [evidence]"
7. **Success metric** — CVR, ATC rate, AOV, scroll depth, upsell take rate
8. **Reference** — Competitor examples, client assets, brand notes, psychological frameworks

## Testing Methodology

### Hypothesis Chain Format
```
IF we [change X on page Y]
THEN we expect [metric] to [increase/decrease] by [range]
BECAUSE [evidence from audit/data/research]

IF that works → THEN test [next iteration]
IF that fails → THEN test [alternative approach]
```

### Test Prioritisation (ICE Framework)
- **Impact** — How much revenue could this move? (1-10)
- **Confidence** — How sure are we it'll work? (1-10)
- **Ease** — How fast can we ship it? (1-10)

Score = (I + C + E) / 3. Anything above 7 ships this month.

### Minimum Test Duration
- 2 weeks minimum or 1,000 visitors per variant (whichever comes first)
- Statistical significance threshold: 95%
- Don't call tests early — let them run
