# Delivery Framework

## Monthly Rhythm

### Week 1: Strategy & Briefs
- Review last month's test results and analytics
- Update roadmap priorities based on data
- Write design briefs for this month's builds
- Client check-in: "Here's what we're building and why"

### Week 2-3: Build & Design
- Design pages per brief (section-by-section)
- Client review and feedback round
- Development and QA
- Pre-launch checklist

### Week 4: Launch & Measure
- Push live
- Set up tracking and monitoring
- Document baseline metrics
- Plan next month's tests based on this month's builds

## Design Brief Template

Each brief should include:
1. **Page type** — Landing page, PDP, cart redesign, etc.
2. **Traffic source** — Where are visitors coming from? (Meta, Google, email)
3. **Objective** — What's the one job this page needs to do?
4. **Audience** — Who lands here? Cold traffic? Retargeting? Existing customers?
5. **Section-by-section breakdown** — Hero, social proof, features, objection handling, CTA
6. **Hypothesis** — "We believe [change] will [outcome] because [evidence]"
7. **Success metric** — CVR, ATC rate, scroll depth, etc.
8. **Reference** — Competitor examples, client assets, brand notes

## Testing Roadmap

### Hypothesis Chain Format
```
IF we [change X on page Y]
THEN we expect [metric] to [increase/decrease] by [range]
BECAUSE [evidence from audit/data/research]

IF that works → THEN test [next iteration]
IF that fails → THEN test [alternative approach]
```

### Test Prioritisation (ICE Framework)
- **Impact** — How much revenue could this move? (1-10)
- **Confidence** — How sure are we it'll work? (1-10)
- **Ease** — How fast can we ship it? (1-10)

Score = (I + C + E) / 3. Anything above 7 ships this month.

## Monthly Reporting

What the client sees each month:
- Pages shipped (with before/after)
- Test results (with statistical significance)
- CVR movement (overall and per page)
- Revenue attributed to changes
- Next month's roadmap priorities
- Recommendations and strategic notes
