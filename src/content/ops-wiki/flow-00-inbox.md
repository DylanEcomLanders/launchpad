# Onboarding Inbox — Intake & Triage

## What This Is

The Onboarding Inbox sits above all four project flows. Every new client starts here — regardless of offer type. Nothing enters a project flow until the intake is complete and the project has been triaged.

This is not a passive waiting area. The inbox must be actively worked. Projects sitting unactioned in intake for more than 48 hours are a failure.

## How Projects Arrive

All intake comes through one of these channels:

| Source | Landing Point | Action |
|--------|--------------|--------|
| Project Kickoff form | Launchpad → new portal draft | Verify details, request access |
| Direct sale (call) | Slack notification from AJ/Dylan | Create portal manually, start intake |
| Referral / warm intro | Email or DM | Create portal, log source for attribution |
| Upsell from existing client | Existing portal | Add new project to portal, start intake |

## Intake Checklist

Before a project can leave the inbox and enter a flow, every item must be confirmed:

### Client Details
- [ ] Client name (company, not contact)
- [ ] Primary contact (name, email, phone)
- [ ] Project type identified → determines which flow it enters
- [ ] Project value confirmed (contract amount)
- [ ] Start date agreed
- [ ] Payment terms confirmed (upfront, 50/50, retainer billing)

### Store Access
- [ ] Shopify admin access (collaborator request sent + accepted)
- [ ] GA4 viewer access (if applicable)
- [ ] Heatmap tool access — Hotjar, MS Clarity (if applicable)
- [ ] Theme access confirmed (which theme, is it customised?)

### Brand Assets
- [ ] Logo files received (SVG + PNG)
- [ ] Brand guidelines / colours / fonts received
- [ ] Product photography or asset folder shared
- [ ] Existing copy or messaging docs (if any)

### Research Started
- [ ] Site walkthrough completed (screenshot key pages)
- [ ] Tech stack identified (BuiltWith scan)
- [ ] Competitor landscape noted (2-3 competitors)
- [ ] Initial observations documented (obvious gaps, quick wins)

## Triage → Flow Assignment

Once intake is complete, assign the project to the correct flow:

| Offer Type | Flow | Trigger |
|-----------|------|---------|
| Design deliverable only (no dev) | **Design Only** | Client wants designs/wireframes, no build |
| Page build (design + dev) | **Design & Development** | Standard page build project |
| Page build + testing/CRO | **Design, Dev & CRO** | Project includes A/B testing or CRO component |
| Full conversion partnership | **Conversion Partnership** | £8K+/mo retainer, team-led roadmap |

## What Happens at Assignment

1. Portal is created in Launchpad (or project added to existing portal)
2. Project enters the assigned flow at Stage 1
3. Phases are set up in the portal (matching the flow's stages)
4. Team is assigned (designer, developer, strategist as needed)
5. Client receives portal access link
6. Kickoff call is scheduled

## Red Flags at Intake

Stop and escalate if any of these appear:

- **No store access after 3 days** — push back: "We can't start without access. Clock hasn't started."
- **Vague scope** — "just make it look good" needs a scoping call before it enters a flow
- **Multiple decision makers** — identify ONE approval person before proceeding
- **No brand assets** — this will stall design. Get at least logo + colours before assigning
- **Payment not confirmed** — do not start work until payment terms are locked

> **Rule:** A project in the inbox is not a project in progress. It's a lead being qualified. Treat it accordingly — verify everything, assume nothing, and don't let it rot.
