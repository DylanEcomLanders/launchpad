SOP 001 — Client Onboarding: Portal Setup

Operations Client-Facing Owner: PM · Est. time: 15 mins per client

Purpose: Every new client gets a branded portal from day one. No PDFs floating in email. No Slack threads with scattered documents. One link, everything in one place.

When to run: Immediately after a sale closes, before the kickoff call. The client should receive their portal link within 24 hours of signing.

Process

1

Open Project Kickoff in Launchpad

Navigate to Project Kickoff (hero button in the sidebar). This is the single entry point for all new projects.

Launchpad → Project Kickoff
2

Fill in client & project details

Client name, project type (Full Page Build, Landing Page, Retainer, etc.), project overview (2-3 sentences), and kickoff date.

3

Add deliverables

Add each deliverable with its type (PDP, Collection Page, Homepage, Advertorial, etc.). The system auto-calculates design and dev timelines from the business settings.

4

Choose: Create New Portal or Add to Existing

If this is a new client, select "Create New Portal". If the client already has a portal (returning client), select "Add to Existing Portal" and choose their portal from the dropdown.

5

Generate

Hit "Generate All". The system creates: Project Roadmap PDF, Scope of Work PDF, Service Agreement PDF (if agreement details filled). These are auto-embedded in the client portal — not separate files.

Launchpad auto-generates docs + portal
6

Review the portal

Click "Preview" to open the client portal. Check: phases are correct, scope matches what was agreed, documents are attached, team is assigned. Fix anything before sharing.

7

Assign the team

In the portal admin, assign designers and developers from the team directory. Set the Slack channel ID if the client has a shared channel.

Portal Admin → Designers / Developers dropdowns
8

Share the portal link with the client

Copy the portal link and send it to the client via email or Slack. This is their single source of truth. All future updates, designs, staging links, and documents live here.

Portal Admin → Copy Link
Critical Rule

Every single project gets a portal. No exceptions. If a client is paying us, they get a portal. PDFs are never sent standalone — they live inside the portal alongside everything else.

For Retainer Clients

When creating the portal, select "Retainer Client" in the creation flow. Set the testing tier (T1/T2/T3). The portal automatically adapts to show weekly testing cadence instead of a linear timeline. Add the Intelligems API key if available.

For Returning Clients

Don't create a second portal. Use "Add to Existing Portal" so all their work lives in one place. The client sees their project history alongside the new project.

Checklist Before Sharing

☐ Client name correct
☐ Project type correct
☐ All deliverables listed
☐ Dates make sense (kickoff, design, dev, launch)
☐ Scope document generated and accurate
☐ Roadmap document generated
☐ Team assigned (designers + developers)
☐ Slack channel ID set (if applicable)
☐ Portal preview checked — looks professional
